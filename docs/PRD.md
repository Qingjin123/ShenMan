# 声声慢 (ShenMan) - 产品需求文档

## 1. 产品概述

### 1.1 产品名称
- **中文名**：声声慢
- **英文名**：ShenMan
- **命名来源**：李清照词牌《声声慢》，寓意"让声音慢下来，沉淀为文字"

### 1.2 产品定位
一款面向中文用户的开源 macOS 语音转文字工具，基于 Qwen3-ASR 模型，提供高精度的中文语音识别、句子级时间戳和简洁的用户体验。

### 1.3 目标用户
- **核心用户**：需要频繁处理中文语音内容的 Mac 用户
- **典型场景**：
  - 会议录音转纪要
  - 讲座/课程转文字稿
  - 播客/访谈转录
  - 视频字幕生成
  - 语音备忘录整理

### 1.4 产品愿景
成为中文 Mac 用户首选的本地语音转文字工具，在中文场景下体验超越 MacWhisper。

## 2. 功能需求

### 2.1 MVP 功能（P0 - 核心功能）

#### 2.1.1 文件导入
- **功能描述**：支持拖放音频/视频文件到应用窗口
- **支持格式**：
  - 音频：MP3, WAV, M4A, FLAC, AAC
  - 视频：MP4, MOV, AVI
- **交互方式**：
  - 拖放文件到指定区域
  - 或点击选择文件
- **反馈**：
  - 显示文件名、时长、大小
  - 格式不支持时给出提示

#### 2.1.2 语音转录
- **核心模型**：Qwen3-ASR-0.6B（MLX 版本）
- **功能要求**：
  - 支持中文普通话识别
  - 支持中文方言（粤语、川渝话等）
  - 支持中英混杂识别
  - 输出带句子级时间戳的文本
- **性能要求**：
  - M1/M2 芯片：1 分钟音频处理时间 < 10 秒
  - 内存占用 < 4GB
- **进度显示**：
  - 实时显示转录进度（百分比）
  - 预估剩余时间

#### 2.1.3 时间戳处理
- **默认输出**：句子级时间戳
- **聚合策略**：
  - 按标点符号（。！？.!?）聚合
  - 按停顿时间（>0.5s）聚合
- **时间戳格式**：`[00:01:23.450 → 00:01:27.890]`
- **显示方式**：
  - 每句话单独一行
  - 时间戳显示在句首或句尾（可配置）

#### 2.1.4 结果展示与编辑
- **文本展示**：
  - 富文本显示（可区分时间戳）
  - 支持滚动查看
  - 支持文本选择、复制
- **文本编辑**：
  - 可直接编辑转录结果
  - 支持撤销/重做
  - 编辑后时间戳保持不变

#### 2.1.5 导出功能
- **支持格式**：
  - **TXT**：纯文本（可选是否包含时间戳）
  - **SRT**：字幕格式（用于视频字幕）
    ```
    1
    00:00:01,200 --> 00:00:03,500
    今天天气不错
    ```
  - **Markdown**：带时间戳的 Markdown 格式
    ```markdown
    - [00:00:01.2 → 00:00:03.5] 今天天气不错
    ```
- **导出选项**：
  - 选择导出路径
  - 选择是否包含时间戳
  - 选择时间戳格式（句首/句尾）

#### 2.1.6 基础设置
- **模型选择**：
  - Qwen3-ASR-0.6B（默认）
  - Whisper-large-v3-turbo（备选，v1.0 实现）
- **语言设置**：
  - 自动检测（默认）
  - 手动指定（中文、英语、粤语等）
- **时间戳设置**：
  - 显示/隐藏时间戳
  - 时间戳位置（句首/句尾）
  - 时间戳精度（秒/毫秒）

## 3. 非功能需求

### 3.1 性能要求
- **转录速度**：
  - 1 分钟音频处理时间 < 10 秒（M1/M2）
  - 10 分钟音频处理时间 < 90 秒（M1/M2）
- **内存占用**：
  - 空闲状态 < 200MB
  - 转录过程 < 4GB
- **启动时间**：冷启动 < 3 秒

### 3.2 兼容性要求
- **操作系统**：macOS 13.0+
- **芯片支持**：
  - Apple Silicon（M1/M2/M3）—— 主力优化
  - Intel Mac —— 基础支持（性能较低）
- **音频格式**：主流音频/视频格式

### 3.3 隐私要求
- **完全离线**：所有处理在本地进行，无网络请求
- **无数据收集**：不收集用户音频、文本数据
- **开源透明**：代码开源，可审查

### 3.4 可用性要求
- **易用性**：无需技术背景，拖放即用
- **容错性**：
  - 格式不支持时给出清晰提示
  - 转录失败可重试
  - 异常退出可恢复进度
- **可访问性**：支持 VoiceOver

## 4. 用户故事

### 4.1 核心用户故事

**故事 1：会议记录**
> 作为一名职场人士，我希望快速将会议录音转为文字纪要，这样我就不用手动听写，可以节省时间。
> - 场景：2 小时会议录音
> - 操作：拖入录音文件 → 等待转录 → 导出为 Markdown
> - 期望：30 分钟内完成，准确率>90%

**故事 2：课程笔记**
> 作为一名学生，我希望将教授的讲座录音转为文字，这样我可以专注于听讲，不用忙着记笔记。
> - 场景：90 分钟课程录音
> - 操作：导入录音 → 转录 → 导出带时间戳的文本
> - 期望：可以定位到特定时间点复习

**故事 3：视频字幕**
> 作为一名视频创作者，我希望为视频生成字幕，这样我可以快速发布到 B 站/YouTube。
> - 场景：10 分钟视频
> - 操作：导入视频 → 转录 → 导出 SRT 字幕
> - 期望：字幕时间轴准确，可直接使用

**故事 4：方言识别**
> 作为一名广东用户，我希望识别粤语内容，这样我就不用手动转录粤语会议/访谈。
> - 场景：粤语会议录音
> - 操作：选择粤语模型 → 转录 → 导出

## 6. 风险与约束

### 6.1 技术风险
- **MLX 生态成熟度**：Qwen3-ASR 的 MLX 版本可能不稳定
- **模型兼容性**：不同模型输出格式不统一
- **性能优化**：大模型在 Mac 上的内存/速度平衡

**应对方案**：
- 优先使用社区验证的模型版本
- 设计抽象层，隔离模型差异
- 提供模型量化选项

### 6.2 产品风险
- **MacWhisper 竞争**：已有成熟产品
- **苹果官方竞争**：系统级集成 ASR

**应对方案**：
- 专注中文优化，差异化竞争
- 开源建立社区，快速迭代

### 6.3 法律风险
- **模型许可**：Qwen3-ASR 的许可限制
- **分发许可**：App 无签名的法律风险

**应对方案**：
- 使用 MIT/Apache 2.0 等宽松许可的模型
- 在 README 中明确说明无签名情况
