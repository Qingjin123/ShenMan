**功能特性**:

- ✅ 取消操作
- ✅ 重置状态

**状态管理**:
```swift
@MainActor
final class BatchProcessingViewModel: ObservableObject {
    @Published var processingState: BatchProcessingState
    @Published var progress: Double
    @Published var completedCount: Int
    @Published var failedCount: Int
    @Published var currentProcessingFile: String
}
```

**处理状态**:
- `.idle` - 空闲
- `.processing` - 处理中
- `.completed` - 完成
- `.failed` - 失败
- `.cancelled` - 已取消

### ViewModel 模式

```

    ├── @StateObject viewModel: BatchProcessingViewModel
    ├── viewModel.addFiles()
    ├── viewModel.startBatchTranscription()

```
用户操作
    │

BatchImportView

BatchProcessingViewModel

    ▼
BatchProcessingService (actor)
    │

## 📝 开发总结

### 成功经验

1. **ViewModel 模式**
   - 状态管理清晰
   - 业务逻辑与 UI 分离
   - 易于测试和维护

2. **组件化设计**
   - 每个视图职责单一
   - 可复用组件（MetaTag、StatView）
   - 遵循 SwiftUI 最佳实践

3. **类型安全**
   - 使用枚举管理状态
   - 强类型的数据模型
   - 编译时检查

### 遇到的问题

1. **UIKit vs AppKit**
   - 问题：ShareSheet 使用 UIKit
   - 解决：使用条件编译，macOS 用 AppKit
   - 教训：注意跨平台兼容性

2. **actor 隔离**
   - 问题：BatchProcessingService 是 actor
   - 解决：ViewModel 封装 actor 调用
   - 教训：合理设计并发架构

3. **sheet 传值**
   - 问题：如何传递 selectedRecord
   - 解决：使用 `@Binding var item: T?`
   - 教训：熟悉 SwiftUI API
