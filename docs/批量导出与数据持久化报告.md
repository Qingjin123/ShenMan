## ✅ 完成的功能

### 1. 批量导出功能

#### BatchExportService (新建)

**文件**: `Services/BatchExportService.swift`

**功能特性**:
- ✅ actor 保证并发安全
- ✅ 批量导出多个转录结果
- ✅ 支持三种格式（TXT、SRT、Markdown）
- ✅ 实时进度回调
- ✅ 错误处理和统计
- ✅ 自动生成文件名
- ✅ 创建导出目录

**导出流程**:
```swift
actor BatchExportService {
    func exportBatch(
        results: [TranscriptionResult],
        format: AppSettings.ExportFormat,
        directory: URL,
        progressHandler: (@Sendable (Double, String) -> Void)?
    ) async throws -> [URL]
}
```

**文件名格式**:
```
原文件名_时间戳。扩展名
例如：会议录音_202603181430.txt
```

- ✅ 添加 `batchExportResults` 数组

**代码**:
```swift
func saveToHistory(result: TranscriptionResult) async {
    let transcript = result.sentences.map { $0.text }.joined(separator: "\n")
    _ = await historyRepository.addHistoryRecord(from: result, transcript: transcript)
}
```

## 📝 开发总结

### 成功经验

1. **actor 并发安全**
   - BatchExportService 使用 actor
   - 避免数据竞争
   - 编译器保证安全

2. **模块化设计**
   - Exporter 协议抽象
   - 批量导出与单文件导出分离
   - 易于扩展新格式

3. **用户体验**
   - 实时进度反馈
   - 清晰的完成提示
   - 一键打开导出目录

### 遇到的问题

1. **Exporter 协议不一致**
   - 问题：协议定义与实际实现不符
   - 解决：修改调用方式，先获取 Data 再写入
   - 教训：保持协议与实现一致

2. **actor 隔离问题**
   - 问题：在 MainActor 中访问 actor 属性
   - 解决：先复制到局部变量再传递
   - 教训：注意 actor 隔离边界

3. **文件路径处理**
   - 问题：macOS 沙盒限制
   - 解决：使用文件选择器获取权限
   - 教训：注意 macOS 沙盒规则
